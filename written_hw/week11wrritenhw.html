  • Manual tests are done by having a user enter data via the UI and test to see that it matches. Automated tests have preset arguments, values, or whatever else is being tested preset by the developer in the test routes. In essence it is a manual test is fluid whereas automated tests are static after they have been served.
  • 16. What is the difference between unit testing, integration testing, and end-to-end testing?
  • Unit tests focus on individual pieces of the program like a function or method. Integration tests check whether multiple components work together properly to achieve a larger action. And End-to-end tests if a program works in an environment made to mimic its full deployment.
  • 17. Can you provide examples of functional, regression, smoke, and performance tests?
  • Functional tests- unit tests that check methods are a form of functional tests. Regression tests- running a test in 2016 to make sure that your var variables are still functioning correctly after the 2015 code drop. Smoke testing- running an automated or manual test on Day 0 to ensure that a new build is stable. Performance test- runs on a program to check to see if it can handle different load sizes put on the program. For example testing servers before the release of a popular game to make sure it doesn't crash when users get their hands on it.
